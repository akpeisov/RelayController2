RS-485
Идея в том, чтобы при старте все рассылают сообщения discovery.
Чтобы все устройства знали кто есть вокруг, т.е. мастера нет, все равнозначные.
Каждое устройство шлет все изменения в шину, т.е. событие и состояние и входов и выходов

Протокол
M T LL SSSSSS DDDDDD II P...
M - magic, always 0xA5
T - type msg_type_t;
LL - payload len
SS - source address
DD - destination address, FF for broadcast
II - message id, autoincrement    
PP - payload, can be null

Конфиг должен храниться на каждом устройстве

Каждому событию нужен уникальный ID, чтобы сервер умел делать дедупликацию:
event_id = <node_uid> + <uptime_ms> + <local_counter>

На сервере:

UNIQUE(event_id)

👉 Даже если 2 узла случайно отправят одно событие — сервер примет только одно.

Это страховка уровня backend, must-have.

Алгоритм

Узел А генерирует событие E

Делает broadcast по RS485:

EVENT { id, source_node, payload }


Если узел А онлайн →
он сам отправляет на сервер и рассылает:

EVENT_ACK { event_id }


Если А офлайн →
каждый узел ждёт случайную задержку (jitter):

node1: 80ms
node2: 120ms
node3: 300ms


Тот, кто первый:

не получил EVENT_ACK

и сам онлайн

→ отправляет событие на сервер
→ рассылает EVENT_ACK

Остальные, увидев EVENT_ACK — отменяют отправку
